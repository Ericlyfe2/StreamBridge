The pipeline

Three steps. No friction.

Drop in a source. The bridge fetches metadata, joins the swarm or pulls the stream, and emits a signed download URL the instant bytes are ready.

01

Source

Submit a magnet link, upload a .torrent, or paste a video URL. The bridge infers the infohash and queues the job in milliseconds.

02

Bridge

libtorrent spins up DHT, peers, LSD. yt-dlp handles URL flows. SSE streams progress, peers, and rate as bytes land — no polling.

03

Stream

On completion, the API returns a time-limited HMAC-signed URL. Plug it straight into a player, a downloader, or a CDN. Done.

Features

Built for real-time delivery.

FastAPI core, libtorrent for swarms, yt-dlp for URLs, SQLite-backed jobs, and HMAC-signed time-limited download URLs. Every detail tuned for flow.

Live SSE progress

Server-Sent Events stream status, peers, seeds, rate, and bytes per second the moment they change.

HMAC-signed URLs

Time-limited tokens scope every download to a single file with a verifiable signature — no leaks.

Tri-source ingest

Magnet, .torrent, or HTTP(S) video URL — one pipeline, one API surface, one signed result.

Pause · resume · cancel

Full control over each torrent handle, surfaced as one-call REST endpoints with safe state transitions.

SQLite-backed jobs

Durable jobs and files tables, ready to swap for Postgres + Redis + Celery when you scale up.

Run anywhere

Pure Python, runs locally with one command or via Docker Compose. Vendor-bundled libtorrent on Windows.
